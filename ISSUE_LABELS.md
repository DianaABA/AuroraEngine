# Issue Labels

Use these labels to categorize issues and PRs.

## Type
- `type:bug` — Something isn’t working
- `type:feat` — New feature or enhancement
- `type:docs` — Documentation-related changes
- `type:refactor` — Code refactor without behavior change
- `type:test` — Testing improvements or coverage
- `type:chore` — Maintenance, infra, tooling

## Priority
- `prio:high` — User-facing impact, needed soon
- `prio:med` — Normal priority
- `prio:low` — Nice to have

## Status
- `status:help-wanted` — Good candidate for new contributors
- `status:needs-info` — More details required
- `status:wip` — In progress

## Area
- `area:vn` — VN runtime & steps
- `area:state` — State, flags, persistence
- `area:docs` — Docs & tutorials
- `area:build` — Build, CI, packaging

Apply the minimal set of labels that clarify intent and scope.
